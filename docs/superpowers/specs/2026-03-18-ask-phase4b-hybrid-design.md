# Ask Phase-4b (Hybrid Mode) Test Design

## Context and Goal

Phase-4 (simple mode) established deterministic `Ask` behavior coverage.

Phase-4b extends `Ask` testing to hybrid retrieval mode (`EnableHybridSearch=true`) to verify that Ask remains correct when memory context comes from hybrid search logic.

## Scope

In scope:

- `Ask` behavior with hybrid search enabled
- Prompt contract validation in hybrid mode
- Error propagation for hybrid Ask path
- Empty-result behavior in hybrid mode
- Source-line rendering in hybrid-mode prompt when `OriginalContent` is present

Out of scope:

- Re-validating full hybrid ranking internals already covered in Phase-3
- `Ask` answer quality semantics
- Production prompt redesign

## Approach

Behavior-first tests with deterministic fixtures and existing fakes.

- Reuse `fakeEmbedder`, `fakeLLMClient`, and `fakeVectorStore`.
- Keep assertions centered on observable Ask behavior (return value, call counts, prompt fragments).
- Avoid tight coupling to internal fusion scores.
- For deterministic hybrid retrieval-error testing, allow a minimal test-only override hook in engine search path (no semantic behavior change in production default path).

## Test Architecture

### Files

- Modify: `core/engine/ask_test.go` (add `TestAskHybrid_*` cases)
- Reuse: `core/engine/test_fakes_test.go`
- Minimal production change if needed: `core/engine/engine.go`, `core/engine/search.go` (hybrid search override hook)

### Principles

- Hybrid mode explicitly enabled in all `TestAskHybrid_*` tests.
- Deterministic embeddings/timestamps to avoid flaky context ordering.
- Prompt assertions use fragment checks (`strings.Contains`) rather than full literal matching.

## Behavior Coverage Matrix

### 1) Hybrid success path

- `Ask` returns LLM response unchanged when hybrid search yields results.
- LLM `Chat` is called exactly once.
- Captured message contract checks:
  - exactly 2 messages
  - role order is `system` then `user`
  - system content is non-empty
  - user content includes required prompt sections

### 2) Hybrid prompt contract

- User prompt includes required fragments:
  - `Memory Context:`
  - numbered entries (`[1]`, optionally `[2]`)
  - `Question: ...`
  - `Answer:`
- At least one hybrid-retrieved memory appears in context text.

### 3) Hybrid source-line behavior

- If retrieved payload contains `OriginalContent`, prompt includes `Source: ...` line.
- This is validated in hybrid Ask flow (not only simple flow).

### 4) Hybrid error propagation

- Query embedder failure in hybrid Ask path is returned by `Ask` unchanged.
- LLM `Chat` is not called on this path.
- Hybrid retrieval failure is returned by `Ask` unchanged (triggered via test-only hybrid-search override hook).
- If hybrid search succeeds but LLM `Chat` fails, Ask returns the same LLM error.

### 5) Hybrid empty-result fallback

- If hybrid search returns no memories, Ask returns exact fallback:
  - `No relevant memories found.`
- LLM `Chat` is not called.

## Determinism and Stability

- Fixed `nowFn` and deterministic vector mappings.
- Stable fixtures to avoid accidental ranking ties where order is asserted.
- No network/external LLM dependency.

## Validation Commands

Required:

- `go test ./core/engine -run TestAskHybrid -v`
- `go test -coverprofile=coverage.phase4b.out ./core/engine`

Recommended:

- `go test ./core/engine -v`
- `go test ./core/utils ./core/retrieval ./core/index ./core/engine`
- `go test ./...`

Coverage artifact:

- `coverage.phase4b.out` generated by required command above

## Success Criteria

- `TestAskHybrid_*` suite added and deterministic.
- Hybrid Ask success/contract/error/empty/source behaviors covered.
- Required command passes: `go test ./core/engine -run TestAskHybrid -v`.
- Phase-4b coverage artifact generated locally.

## Risks and Mitigations

- Risk: brittle prompt checks.
  - Mitigation: fragment-level assertions only.

- Risk: re-testing hybrid internals redundantly.
  - Mitigation: focus on Ask integration behavior rather than score internals.

- Risk: accidental simple-mode leakage.
  - Mitigation: each `TestAskHybrid_*` enforces `EnableHybridSearch=true` in setup.
